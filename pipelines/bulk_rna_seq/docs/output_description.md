# Bulk RNA-seq Output Description

## Gene-Level Salmon Workflow Outputs

| Output | Description |
| --- | --- |
| FastQC HTML reports | Per-FASTQ sequencing quality reports |
| MultiQC report | Combined QC summary |
| Trimmed FASTQs | Optional reads after trimming |
| Cutadapt logs | Trimming summary and retained-read counts |
| Salmon sample directories | Per-sample quantification outputs |
| `quant.sf` | Main transcript-level Salmon output |
| `salmon_quant_files.tsv` | Helper file listing sample IDs and `quant.sf` paths |
| `sample_metadata_template.csv` | Editable metadata starter generated from Salmon output folders |
| Gene-level count matrix | Matrix generated by `tximport` |
| DESeq2 object or saved RDS | Reusable statistical analysis object |
| Differential expression tables | Results for planned contrasts |
| Significant gene tables | Filtered DE genes with thresholds documented |
| PCA plots | Sample clustering and outlier review |
| Heatmaps | Variable or significant expression patterns |
| Enrichment outputs | GO, KEGG, Hallmark, or GSEA results |
| R Markdown or HTML report | Reproducible summary of analysis |

## TE-Expression Outputs

| Output | Description |
| --- | --- |
| STAR logs | Alignment summary and mapping metrics |
| STAR sorted BAM files | Genome-aligned reads for TE analysis |
| BAM indexes | Index files for BAM inspection |
| TElocal `.cntTable` files | TE count tables for each sample |
| TE count matrix | TE-by-sample matrix imported into R |
| TE DESeq2 result tables | Differential TE-expression contrasts |
| TE PCA plots | Sample-level TE-expression QC |
| TE heatmaps | TE-expression patterns |
| TE class summaries | Family/class/superclass summaries |

## Files That Should Not Be Committed

Do not commit generated FASTQs, BAMs, indexes, Salmon outputs, count matrices, plots, reports, or private project results to this template repository.

## Script-Generated Output Locations

The script templates use editable `OUTPUT_DIR` variables rather than fixed repository paths. A typical project may use:

```text
/path/to/output/
  salmon_hg38/
  salmon_mm10/
  metadata/
  star_mm10/
  telocal_mm10/
```

These project outputs should stay in the project workspace, not in this repository.
